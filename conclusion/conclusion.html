Evaluation

This project demonstrates a working wildfire risk monitoring system that integrates sensors, a microcontroller, a server, and a machine learning model. The system collects temperature and humidity data using a Raspberry Pi Pico, sends the data to a FastAPI server, and generates a wildfire risk prediction. Developing this project gave me experience in embedded programming, backend development, and integrating machine learning into a real-time system.

Limitations:

  • Dataset limitations
    The model was trained using a Californian wildfire dataset, which does not represent environmental conditions in other regions, therefore the predictions may not be fully accurate outside of similar climates. But this could be easily improved by training the model on a more diverse dataset that includes data from various regions and climates.

  • Limited input features
    The model only uses temperature and humidity, while real wildfire risk is also affected by factors such as wind speed, vegetation type, and soil moisture. This could be improved by adding more sensors.

  • Wi-Fi dependency
    The emergency alert button sends signals over Wi-Fi, which may not be available in remote forest environments. I could further improve it by deploying a network of microcontrollers that communicate with each other. (My initial idea. But I only had access to one microcontroller so I couldn't test it).

For me, building the website was the most enjoyable part of the project. It was refreshing to work without frameworks like React (due to the project brief) or Bootstrap and instead build everything from scratch. This reminded me that programming, at its core, is simply about building things that work.

Overall, I had a lot of fun working on the project as it provided me with valuable insight into the challenges of building a real-world system, while also allowing me to learn new skills and technologies along the way.
